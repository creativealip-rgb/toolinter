# Panduan Penulisan Artikel Toolinter.net — SEO, Helpful Content, AEO/GEO

**Project:** Toolinter.net  
**Domain:** https://toolinter.net  
**Tujuan:** membuat artikel tutorial dan utilitas yang bisa ranking di Google, muncul sebagai jawaban cepat, dan mudah dikutip oleh AI answer engines.  
**Audience utama:** pengguna Indonesia yang butuh menyelesaikan tugas praktis: membuat surat, resize foto, menghitung gaji/pajak, kompres PDF, membuat CV, menghitung kebutuhan UMKM/keuangan.  
**Last updated:** 26 Juni 2026

---

## 1. Prinsip besar Toolinter

Toolinter bukan blog opini. Artikel Toolinter harus membantu user menyelesaikan pekerjaan **sekarang** dengan tool gratis.

Setiap artikel harus menjawab:

1. User mau menyelesaikan tugas apa?
2. Syarat/aturan apa yang perlu diketahui?
3. Langkah praktisnya apa?
4. Tool Toolinter mana yang dipakai?
5. Kesalahan apa yang harus dihindari?
6. Output seperti apa yang benar?

### Positioning konten

- Cepat.
- Praktis.
- Indonesia-first.
- Tool-led.
- Aman dan jelas.
- Tidak bertele-tele.

---

## 2. Pegangan dari riset SEO + AI search terbaru

Berdasarkan Google Search Central dan tren AEO/GEO 2025–2026:

1. **Helpful content menang.** Artikel harus menyelesaikan masalah user, bukan cuma mengejar keyword.
2. **AI Overviews/AI Mode pakai fondasi Search yang sama.** Tidak perlu markup khusus AI; yang penting crawlable, indexable, konten bermanfaat, struktur jelas, dan data/schema sesuai isi halaman.
3. **Answer-first content lebih mudah dikutip.** Untuk AI search, bagian jawaban langsung 40–70 kata sangat penting.
4. **Structured content membantu mesin memahami halaman.** Gunakan heading, bullet, step-by-step, FAQ, dan schema yang sesuai.
5. **Trust penting untuk topik sensitif.** Artikel tentang gaji, pajak, pinjol, dokumen kerja, CPNS, atau hukum harus menyebut batasan dan menyarankan cek sumber resmi jika diperlukan.

---

## 3. Definisi SEO, AEO, GEO untuk Toolinter

- **SEO:** artikel ditemukan dari keyword seperti “cara membuat surat resign”, “kompres PDF online”, “cara hitung gaji bersih”.
- **AEO:** artikel bisa menjawab langsung pertanyaan user di snippet/AI Overview.
- **GEO:** artikel bisa dipakai AI sebagai sumber jawaban karena isinya jelas, terstruktur, dan punya fakta/step yang dapat diverifikasi.

---

## 4. Standar minimum publish/index Toolinter

Artikel boleh index jika memenuhi:

- Minimal **900 kata** untuk tutorial biasa.
- Minimal **1.200–1.800 kata** untuk topik YMYL-ish: pajak, gaji, pinjol, CPNS, hukum/dokumen kerja.
- Jawaban langsung di awal 40–70 kata.
- Langkah penggunaan tool jelas.
- Minimal 1 CTA ke tool terkait.
- Minimal 2 internal link ke tool/artikel terkait.
- FAQ minimal 3 pertanyaan.
- Ada disclaimer untuk topik legal/finance/government.
- Metadata lengkap.
- Tidak mengklaim sebagai pengganti konsultan hukum/pajak/instansi resmi.

Jika belum memenuhi, jadikan draft/noindex.

- Definisi surat.
- Kapan dipakai.
- Struktur surat.
- Contoh lengkap.
- Kesalahan umum.
- CTA ke generator surat.
- Disclaimer: template perlu disesuaikan dengan kebijakan perusahaan/instansi.

Contoh CTA:

```md
Mau lebih cepat? Gunakan generator surat Toolinter untuk membuat draft surat otomatis, lalu cek ulang nama, tanggal, dan tujuan surat sebelum dikirim.
```

- Batas ukuran file jika relevan.

- Contoh hitung.

- CTA ke kalkulator.

### D. PDF & Converter

Wajib ada:
- Kapan dipakai.
- Cara pakai tool.

### E. CV & Lamaran

Wajib ada:
- Tujuan dokumen.
- Struktur ATS/HR-friendly.
- Contoh isi.
- Kesalahan umum.
- CTA ke CV/cover letter generator.

### F. UMKM & Bisnis

Wajib ada:
- Rumus/konsep.
- Contoh angka.
- Interpretasi hasil.
- Kesalahan pricing/margin.
- CTA ke kalkulator.

Wajib ada:
- Sumber aturan/rujukan resmi jika memungkinkan.
- Tanggal update.
- Disclaimer: cek pengumuman resmi.

## 8. SEO checklist Toolinter

### Keyword

- Focus keyword harus mewakili task, bukan topik umum.
- Contoh bagus:
  - `cara membuat surat resign`
  - `ukuran foto CPNS 2026`
  - `kompres PDF online`
  - `cara hitung gaji bersih`
- Contoh kurang bagus:
  - `surat`
  - `PDF`
  - `gaji`

### Title

Format:

```txt
Cara [Task] [Konteks] ([Tahun]) + [Manfaat]
```

Contoh:

```txt
Cara Membuat Surat Resign yang Sopan dan Profesional [2026]
```

### Meta title

- 45–60 karakter.
- Masukkan keyword utama.
- Jangan terlalu panjang.

### Meta description

- 120–160 karakter.
- Jelaskan hasil yang user dapat.
- Sebut gratis/online jika relevan.

Contoh:

```txt
Panduan cara membuat surat resign yang sopan, lengkap dengan struktur, contoh, tips, dan generator surat gratis di Toolinter.
```

## 9. AEO/GEO checklist Toolinter

### Answer block di awal

Setiap artikel harus punya jawaban ringkas yang bisa dikutip.

Contoh:

```md
Surat resign yang baik harus mencantumkan identitas karyawan, tujuan surat, pernyataan mengundurkan diri, tanggal efektif resign, ucapan terima kasih, dan tanda tangan. Gunakan bahasa sopan, hindari menyalahkan perusahaan, dan serahkan sesuai aturan notice period di kontrak kerja.
```

### Step-by-step harus eksplisit

AI dan user suka langkah yang jelas:

```md
1. Buka tool Kompres PDF.
2. Upload file PDF.
3. Pilih tingkat kompresi.
4. Klik Kompres.
5. Download hasil.
6. Buka file hasil untuk memastikan kualitas tetap terbaca.
```

### Fakta harus terstruktur

Gunakan bullet/tabel markdown untuk:
- ukuran file
- format dokumen
- komponen biaya/gaji
- syarat upload
- perbandingan pilihan

### Snippet-ready section

Setiap artikel minimal punya 3 section yang bisa berdiri sendiri sebagai jawaban:

- “Apa itu ...?”
- “Cara ...”
- “Syarat ...”
- “Kesalahan umum ...”
- “FAQ ...”

## 10. E-E-A-T dan trust Toolinter

Toolinter adalah tool site. Trust dibangun lewat:

- transparansi proses: file diproses di browser jika benar secara teknis.
- contoh output/simulasi.
- tanggal update.
- disclaimer untuk topik legal/finance/government.
- link ke sumber resmi jika menyebut aturan.
- tidak overclaim.

### Disclaimer standar

Untuk surat/legal:

```md
Catatan: Template ini bersifat panduan umum. Sesuaikan isi surat dengan kebijakan perusahaan/instansi dan kebutuhan pribadi sebelum digunakan.
```

Untuk pajak/gaji:

```md
Catatan: Simulasi ini bersifat estimasi. Tarif pajak, BPJS, UMR, dan aturan ketenagakerjaan bisa berubah; cek sumber resmi atau konsultasikan dengan HR/ konsultan pajak untuk keputusan final.
```

Untuk CPNS/government:

```md
Catatan: Syarat dokumen dapat berubah mengikuti pengumuman resmi instansi. Selalu cek portal resmi sebelum mengirim berkas.
```

## 12. Rubrik skor artikel Toolinter

Total 100 poin:

- Intent match/task completion: 20
- Jawaban langsung dan step-by-step: 15
- Akurasi fakta/rumus/syarat: 15
- Contoh/simulasi/output: 15
- CTA ke tool relevan: 10
- Metadata dan keyword: 10
- FAQ dan AEO/GEO extractability: 10
- Trust/disclaimer/sumber: 5

### Keputusan

- **85–100:** publish + index.
- **70–84:** publish setelah review minor.
- **50–69:** noindex/backlog, perlu expand.
- **<50:** draft ulang.

## 13. Workflow produksi Toolinter

1. Pilih tool atau kategori target.
2. Tentukan task keyword: “cara ...”, “ukuran ...”, “contoh ...”, “hitung ...”.
3. Buat outline sesuai kategori.
4. Tulis jawaban langsung di awal.
5. Tambahkan syarat/rumus/contoh.
6. Tambahkan langkah pakai Toolinter.
7. Tambahkan disclaimer bila topik sensitif.
8. Tambahkan FAQ.
9. Lengkapi metadata.
10. Skor dengan rubrik.
11. Publish/index hanya jika skor 85+.
